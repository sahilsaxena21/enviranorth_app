# How to Add a New PDF to the Wiki

This guide walks you through adding a new Envira-North PDF to the wiki using Claude Code (Opus). Each step includes the exact prompt to paste into Claude Code.

**Prerequisites:** Claude Code installed and open in the `climtek` project folder.

---

## Step 0 — First-Time Setup (Clone Repo & Python Environment)

Skip this step if you've already cloned the repo and set up the Python environment.

### 0a — Clone the Repository

```bash
git clone https://github.com/sahilsaxena21/climtek_enviranorth_wiki.git climtek
cd climtek
```

### 0b — Repo Structure Overview

```
climtek/
├── raw_pdfs/                  # Drop new Envira-North PDFs here (Step 1)
├── raw_text/                  # Extracted text files (generated by Step 2)
├── extract_pdf_pdfplumber.py  # PDF text extraction script
├── requirements.txt           # Python dependencies
├── wiki_audits/               # Audit reports (generated by Step 5)
│
└── wiki/                      # ← The wiki itself
    ├── CLAUDE.md              # Rules & conventions for wiki editing
    ├── index.md               # Master index of all wiki pages
    ├── glossary.md            # Envira-North terminology
    │
    ├── products/              # One file per product (Sailfin sizes, Jazz, Alite, etc.)
    ├── controls/              # One file per control system (SmartAIR, LVC, ZoneAIR, etc.)
    ├── guides/                # Installation, maintenance, warranty guides
    ├── comparisons/           # Side-by-side product/control comparisons
    ├── applications/          # Application-specific pages (dairy, warehouse, etc.)
    └── image-descriptions/    # Text descriptions of images from source PDFs
```

### 0c — Set Up the Python Environment

Paste this prompt into Claude Code:

```
I need you to set up a Python environment for this project. Here's what to do:

1. Check if Python 3.9 or higher is installed on my system. If not, tell me how to install it for my OS.
2. Create a virtual environment in this project folder: python -m venv venv
3. Activate the virtual environment.
4. Install the dependencies from requirements.txt: pip install -r requirements.txt
5. Verify the install by running: python -c "import pdfplumber; import docx; print('OK')"

Tell me if anything goes wrong.
```

---

## Step 1 — Add the PDF

Manually copy your new PDF file into the `raw_pdfs/` folder. Use the original filename from the Envira-North website — do not rename it.

Once the file is in `raw_pdfs/`, confirm with Claude Code:

```
I just added a new PDF to raw_pdfs/. Please run: ls raw_pdfs/ and confirm you can see the new file.
```

---

## Step 2 — Extract Text from the PDF

Paste this prompt into Claude Code:

```
Run the PDF text extraction script:

python extract_pdf_pdfplumber.py

This script automatically skips any PDF that has already been extracted, so it will only process the new file. Confirm that it produced a new .txt file in raw_text/ and tell me how many words and tables it extracted.
```

---

## Step 3 — Plan the Wiki Changes (Review Before Any Edits)

This is the most important step. Claude will read the extracted text, figure out what it contains, and propose changes for your review — **without making any edits yet**.

Paste this prompt into Claude Code:

```
I just added a new PDF and extracted its text. Read the CLAUDE.md file in the wiki/ folder first to understand the wiki rules and structure. Then:

1. Read the new .txt file in raw_text/ (the most recently created one).
2. Identify what this document is about:
   - Product name, SKU(s), product line
   - Is this a new product, a new control, a new guide, or an update to an existing wiki page?
   - Key specs, dimensions, compatibility info, or procedures found in the text
3. Read index.md to see what already exists in the wiki.
4. If this updates an existing page, read that page so you can compare.

Then present me with a plan:
- Which wiki file(s) you will create or update
- A summary of the key facts you will add (specs, compatibility, procedures, etc.)
- Any gaps or unclear content in the source text (OCR issues, missing data)
- Any other wiki pages that may need cross-reference updates (index.md, related products, controls, comparisons, applications)

DO NOT make any changes yet. Just present the plan for my review.
```

Review the plan. Tell Claude what to change, add, or remove. Once you're satisfied, move to Step 4.

---

## Step 4 — Make the Changes

Once you've reviewed and approved the plan from Step 3, paste this prompt:

```
I approve the plan. Go ahead and make the changes. Follow all the rules in wiki/CLAUDE.md:
- Use the correct frontmatter template for the file type (product, control, guide, comparison, or application)
- List the exact raw_text filename in source_docs
- Set last_verified to today's date
- Update index.md if you created a new page
- Update cross-references (controls_compatible, works_with, related, etc.) on any affected files
- Use explicit size-specific slugs (e.g., sailfin-geared-24ft) in works_with fields, not generic family slugs
- Do not invent any specs — if something is unclear, use an Unverified callout
```

---

## Step 5 — Run a Wiki Audit

After the changes are made, run an audit to catch any issues. Paste this prompt:

```
Run a full wiki audit following the audit workflow in wiki/CLAUDE.md. Write the audit report to wiki_audits/audit-YYYY-MM-DD.md using today's date. Check for:
- Frontmatter problems
- Broken source_doc references
- Broken cross-reference slugs
- Bidirectional relationship mismatches
- Missing index entries
- Unverified claims
- Cross-wiki conflicts

Give me the summary when done.
```

---

## Step 6 — Review Audit Recommendations Together

Once the audit is complete, go through the findings with Claude one at a time. Paste this prompt:

```
I want you to go over your recommendations from the audit one by one and I'll review — accept or decline — one by one. Let's go over this together one by one. Present the first recommendation, explain what the issue is and what the fix would be, then wait for my response before moving to the next one.
```

For each recommendation, you can say:
- **Accept** — Claude will make the fix
- **Decline** — Claude will skip it and move on
- **Skip / flag for follow-up** — Claude will note it and move on (useful when you need to check with Envira-North first)

---

## Step 7 — Push Changes to GitHub

After all changes are finalized, push to the shared wiki repo:

```
Commit all wiki changes and push to origin. Use a descriptive commit message summarizing what was added or updated.
```
